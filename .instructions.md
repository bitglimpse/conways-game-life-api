# Conway's Game of Life API - Agent Context

## Project Overview

This is a production-ready RESTful API implementing Conway's Game of Life in C#/.NET 8.

**Always reference [REQUIREMENTS.md](REQUIREMENTS.md) for the original project requirements and objectives.**

## Key Architectural Decisions

1. **No Authentication/Authorization** - By design per requirements
2. **PostgreSQL + JSONB** - Efficient board state storage
3. **Cycle Detection** - SHA256 hashing to detect static and oscillating patterns
4. **On-Demand Computation** - No caching, compute fresh each request
5. **Controller-based API** - Not Minimal API, for better structure and testability

## Code Conventions

- **Layered Architecture**: Controllers → Services → Repositories → Database
- **Dependency Injection**: Use constructor injection for all dependencies
- **Async/Await**: All I/O operations are asynchronous
- **FluentValidation**: Input validation with clear error messages
- **Repository Pattern**: Abstract data access through interfaces
- **Domain Exceptions**: Custom exceptions for business logic errors

## Common Tasks

### Adding New Endpoint
1. Add method to `IBoardService` interface
2. Implement in `BoardService` with business logic
3. Add controller action in `BoardsController`
4. Add DTOs if needed in `Models/DTOs/`
5. Add validation if needed in `Validators/`
6. Write unit tests for service logic
7. Write integration tests for full HTTP flow

### Modifying Game Logic
- Edit `ConwayGameEngine.cs` for rule changes
- Update tests in `ConwayGameEngineTests.cs`
- Ensure all 27 engine tests still pass

### Database Changes
1. Modify entity in `Models/Domain/`
2. Update `ApplicationDbContext` if needed
3. Create migration: `dotnet ef migrations add MigrationName`
4. Update repository if new queries needed
